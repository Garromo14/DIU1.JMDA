# DIU - Practica 4, entregables

## 1. Migración a React y publicación.

En este apartado, buscamos de alguna forma dotar de funcionalidad a nuestro diseño en figma. Se pretende pasar de diseño a la base para una aplicación en react. En la práctica anterior, definimos un design system que usamos para definir layouts responsive de nuestras diferentes páginas del sitio web. Estos layouts generados en forma de frames se seleccionarán para generar una url de cara a la evaluación en la práctica 5. 


Para ello en un inicio se hizo uso del plugin **locofy**. Este plugin generaba código react y lo integraba en un proyecto definido. Hubo ciertas complicaciones porque los diseños generados no eran los esperados. Este plugin requería seleccionar un frame para convertir a código. Una vez convertido a código, generaba una vista previa del frame y el código de cada elemento. Después, había que integrar el código en nuestro proyecto mediante la ejecución de **locofy pull**. Con esto, la IA de locofy generaba el código en nuestro proyecto. Como hemos comentado, esto acabó dando problemas, pues el resultado era desigual al frame en posición, tamaños, simulación de navegación.

Es por esto que se preguntó al profesor si mantener este diseño o qué hacer y se buscó otra opción. La opción propuesta fue mostrar el prototipo y compartir el enlace generado para su publicación. Este fue el resultado:
